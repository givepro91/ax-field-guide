이 글도 아직 **초안**이다. [Claude Code 글](/guide/claude-code-ax)과 같은 틀을 Codex에 적용하되, 실제 사용 판단은 직접 써본 뒤 채운다.

AX 관점의 질문은 도구가 바뀌어도 같다. [에이전트를 모는 법](/guide/driving-agents)에서 세운 역할·한계·언제 쓰나의 틀로 Codex를 본다. 중요한 건 "어느 도구가 더 똑똑한가"가 아니라 어떤 작업 성격에서 무엇이 더 맞는지인데, 그 비교는 [별도 글](/guide/cc-vs-codex)로 따로 두고 여기서는 Codex 자체의 역할과 한계만 정리한다.

<blockquote class="draft-note"><b>채울 자리.</b> Codex를 실제로 어떤 작업에 썼는지, Claude Code와 강점이 갈린 지점, 승인·검증을 어디 뒀는지가 들어간다. 안 써본 채로 비교 결론을 내지 않는다.</blockquote>

## 채울 항목 (자기 메모)

- 어떤 작업에 Codex가 더 맞았나 / 덜 맞았나
- Claude Code와 다르게 느낀 점 (속도·맥락·도구 연결)
- 승인·검증을 어디에 뒀나
- 안 맞았던 작업
