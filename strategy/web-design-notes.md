# 웹 디자인 노트 (SoT)

> last updated: 2026-06-22 · 성숙도: 정리 중(budding)

AX Field Guide 웹의 디자인·구조 정본. 프로토타입은 `prototype/index.html`이고 로컬(`http://localhost:4321`)에서 본다.

## 한 줄 원칙

**문서형 위키 + 프레젠테이션.** 차분한 기술 위키처럼 읽히되, 같은 문서를 버튼 하나로 강의용 슬라이드로 전환할 수 있다. AI SaaS 랜딩페이지가 아니라 "정리된 지식 베이스" 인상.

## 채택 경위 (중요)

- 처음 제안한 **먹/종이(ink-on-paper) serif·Tufte 방향은 폐기**했다 — 차분하지만 이 프로젝트엔 너무 정적/올드했다.
- 대신 사용자가 제시한 레퍼런스("Clean Wiki — 클린 블루 + Pretendard, 카드 UI + 위키 자산")의 디자인 시스템을 **그대로 채택**한다. 사용자 판단이 우선이고, 카드 UI도 이 기준으로 OK(이전 노트의 "카드 UI 회피"는 무효).

## 디자인 토큰 (정본)

```
--bg:#f7f8fa  (쿨 그레이 배경)   --card:#fff
--ink:#18181b  --sub:#52525b  --mute:#8a93a0
--accent:#2563eb (클린 블루)  --accent-ink:#1d4ed8  --accent-weak:#eef4ff  --accent-border:#cfe0ff
--line:#e5e7eb  --chip:#edeff2   --ok:#047857 / bg #e7f8f1 (성공/라이브 표시)
sans: -apple-system, "Apple SD Gothic Neo", "Pretendard", system-ui …   (Pretendard 기조, 한글 우선)
mono: ui-monospace, Menlo …  (메타·라벨·코드·번호)
```

- 카드: radius 12–16px, 소프트 섀도(`0 1px 2px`, `0 4px 16px` rgba). 라이트 단독(다크 미구현).
- 액센트는 **블루 하나만**. 링크·활성 네비·인포박스 헤더·성숙도 pill·진행바에만 절제해서.

## 레이아웃 (3-pane 위키)

```
[ 좌: 색인 nav ] [ 중앙: 본문 ] [ 우: TOC + 동향 피드 ]
```

- **폭/여백**: shell max-width 1340, 좌우 거터 좁게(clamp 14–24px), 칼럼 gap 30. (콘텐츠를 넓게.)
- **좌측 색인**: 카테고리별 접기(`<details>`)+개수 배지. 아이템은 **한 줄 고정**(말줄임)·배지(예정/자동) 분리 — 줄넘김 금지. 폭 238px.
- **검색**: 상단 검색창이 색인을 실시간 필터(콘텐츠 많아져도 탐색 가능). 결과 0이면 "검색 결과 없음".
- **인포박스**(위키 자산): 본문 우측 카드 — 분류 / 마지막 갱신 / 성숙도(pill) / 관련 문서.
- **사이드 TOC**(우): 스크롤 따라 활성 섹션 하이라이트. ≤1120px에서 숨김.
- **변경 로그**(위키 자산): 본문 하단 카드 — dated 이력.
- 반응형: ≤1120 우측 숨김(2-pane), ≤780 단일 칼럼(nav 상단).

## 콘텐츠 시각 요소 (필수)

모든 문서에 이해를 돕는 그림 최소 1개. 프로토타입 데모: ① 비교 카드(도입↔AX) ② 5계층 스택. 상세 규칙은 `CLAUDE.md`의 "콘텐츠 시각 요소".

## 프레젠테이션 모드

- 우상단 **프레젠테이션** 버튼 → 전체 오버레이 슬라이드. 커버/마무리 슬라이드는 블루 풀블리드로 강조.
- 조작: `←/→`·Space·클릭 이동, `F` 전체화면, `Esc` 종료. 상단 진행바 + 하단 카운터/이전/다음.
- 현재 슬라이드는 수기. **향후**: 문서 섹션에서 자동 생성하거나 `---` 구분자 규칙으로 추출. 핵심 그림은 슬라이드로 재사용.

## 동향(Trends) — 매일 자동 뉴스

매일 AX 기술 동향을 모아 피드로 보여준다(프로토타입: 좌 nav "동향" 그룹 + 우측 "최근 동향" 위젯).

파이프라인(구현됨): **소스(RSS/Atom) → 크롤 → URL 중복 제거 → 요약 → `content/trends/{index,YYYY-MM-DD}.json` → 피드 렌더.** GitHub Action cron 매일 1회 → push → Vercel 재배포. (`scripts/collect-trends.mjs`, `trends-sources.json`, `.github/workflows/trends.yml`)

- **무료 기본**: LLM 미사용, 피드 발췌를 요약으로 → API 키·비용 0. 한국어 LLM 요약은 `--summarize`(`TRENDS_SUMMARIZE=1`) 옵션(유료).
- 소스(확정): Anthropic SDK · Claude Code · OpenAI · MCP servers · Simon Willison.
- 발행: 자동(외부 공개 기사만). 비용 0(무료 모드) · GitHub Actions 무료 사용량.
- **공개 위험**: 출처 링크+발췌만. 회사 내부 식별 정보 유입 금지(게이트 통과 필수).

## 스케일 아키텍처

문서가 많아지면 손수 HTML 유지 불가. `content/**.md`(frontmatter: 분류·갱신일·성숙도·태그·public_risk)를 읽어 **페이지·네비·검색 인덱스·TOC·슬라이드를 생성**하는 정적 사이트 생성기(예: Astro/Eleventy)로 전환. 프로토타입은 그 목표 UI를 검증하는 용도일 뿐.

## 미해결 / 취향
- 슬라이드 자동 생성 방식(섹션 자동 vs `---` 구분자).
- 동향 자동화 주기·소스·발행 게이트.
- 검색을 클라이언트 인덱스(예: 정적 JSON) vs 생성기 내장 중 무엇으로.

## 참고
- 디자인 시스템: 사용자가 제시한 "Clean Wiki(클린 블루 + Pretendard)" 레퍼런스.
- 동향 파이프라인: 사내 유사 뉴스 자동화 사례의 일반 패턴(소스→크롤→중복제거→AI요약→저장→피드). 회사 고유 구현·식별 정보는 본 프로젝트에 반영하지 않음.
